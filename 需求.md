## 1 引言

### 1.1 编写目的

本文档旨在定义“声临其境——基于AIGC的有声书多角色语音合成与情感演绎系统”的软件需求规格。文档详细描述了系统的功能需求、性能需求、接口需求和设计约束，为后续的设计、开发、测试和验收提供依据。

**预期读者**：
- 项目开发团队（AI工程师、前端工程师、后端工程师）
- 项目管理人员
- 测试人员
- 实训评审导师

### 1.2 项目背景

有声书市场近年来呈现爆发式增长，但传统有声书制作面临制作成本高昂（每分钟200-500元）、制作周期漫长（3-6个月）、角色演绎单一、情感表达平淡、产能严重不足以及版权风险高等诸多困境。

随着AIGC技术成熟，特别是情感语音合成技术取得重大突破，声音克隆技术日趋成熟，AI辅助制作可使有声书成本降低90%以上。本项目将AIGC能力与多角色语音合成深度结合，通过大模型自动分析文本角色和情感，一键生成专业级多角色有声书，让每个人都能拥有"千人千声"的专属有声书创作能力。

### 1.3 项目范围

|--------|------|
| **项目名称** | 声临其境——基于AIGC的有声书多角色语音合成与情感演绎系统 |

## 2 总体描述

### 2.1 产品视角

**核心价值主张**：AI赋能，让每一本书都能"声"动人心。

**For** 有声书爱好者、文字内容创作者、出版机构
**Who** 渴望将文字内容转化为高质量有声作品，但受限于成本和专业能力
**The 声临其境** is a AI驱动的多角色语音合成与情感演绎平台
**That** 能自动分析文本角色、智能识别情感、调用不同音色合成，最终输出专业级多角色有声书
**Unlike** 传统有声书制作（成本高、周期长、角色单一、情感平淡）
**Our product** 将AI语音技术与情感计算融合，实现"上传文本 → AI角色分析 → 一键合成 → 专业级多角色有声书"的全新体验

#### 2.1.1 系统架构概述

```
用户端（Web平台）
  ├── 首页：作品广场 + 推荐发现
  ├── 作品创作（核心）
  │   ├── 文本上传/粘贴
  │   ├── 角色分析（AI识别角色、性格、关系）
  │   ├── 音色配置（选择/克隆/定制音色）
  │   ├── 情感标注（自动+手动调整）
  │   └── 合成预览 + 导出下载
  ├── 音色市场：预设音色库 + 克隆音色管理
  ├── 作品管理：我的作品 + 收藏 + 历史
  └── 个人中心：账户管理 + 消费记录

运营管理端（Web后台）
  ├── 作品管理：审核/发布/下架
  ├── 音色管理：音色库维护 + 克隆管理
  ├── 用户管理：权限/订阅/行为分析
  ├── 数据看板：合成量/用户量/热门内容
  └── 内容运营：专题活动 + 推荐配置

AI服务层
  ├── 文本分析Agent（角色识别 + 情感分析）
  ├── 情感语音合成Agent（CosyVoice + 多情感控制）
  ├── 声音克隆Agent（音色定制 + LoRA训练）
  ├── 音频后期Agent（降噪 + 均衡 + BGM）
  ├── 背景音乐Agent（智能匹配 + 淡入淡出）
  └── 质量评估Agent（MOS评分 + 问题检测）

数据层
  ├── 语音数据库（AISHELL-3 + Baker TTS）
  ├── 音色模型库（预设音色 + 用户克隆音色）
  ├── 文本语料库（角色对话 + 情感标注）
  ├── 背景音乐库（场景BGM分类）
  └── 用户行为日志
```

### 2.3 用户特征

#### 普通用户（创作者）

1. **角色**：有声书爱好者、自媒体创作者、出版机构人员、教育内容生产者
2. **能力**：具备基本的网页操作能力，会上传文本文件，能理解音色选择等基本概念
3. **经验**：无需专业音频制作经验。有听有声书或使用音频平台的经验最佳
4. **意愿**：希望以低成本、高效率的方式将文字内容转化为高质量有声作品

#### 管理用户

1. **角色**：平台运营管理人员
2. **能力**：具备基本的Web后台操作能力，了解内容审核规范
3. **经验**：有内容平台或在线服务管理经验最佳
4. **意愿**：高效管理平台内容、用户和音色资源，通过数据分析优化运营策略

### 3.2 子功能模块一 —— 用户管理与认证（M1/M4）

#### 功能简介

提供用户注册、登录、个人信息管理、认证与授权等基础用户功能。

#### 3.2.1 Functional Requirements 1 —— 账户注册与登录

**1. 介绍**

用户可通过手机号或邮箱注册账户，或使用第三方平台（微信/支付宝）快捷登录。登录成功后获取JWT令牌，后续请求需携带令牌进行身份验证。

**预期的错误输入或非法条件：**

1. 用户输入的手机号格式不正确：提示手机号格式错误
2. 用户输入的密码不符合规范：提示密码必须为8-16位，包含数字和字母
3. 手机号已被注册：提示该手机号已注册，可直接登录
4. 用户名或密码错误：提示用户名或密码错误，请重试
5. 网络通信失败：提示网络异常，请检查网络连接后重试

**2. 输入**

A. 对该功能所有输入数据的详细描述

| 数据项 | 输入来源 | 数量 | 度量单位 | 有效输入范围 |
|-------|---------|------|---------|-------------|
| 手机号 | 用户输入 | 每个用户一个 | 条 | 11位中国大陆手机号 |
| 邮箱 | 用户输入 | 每个用户一个 | 条 | 符合邮箱格式，最长64字符 |
| 密码 | 用户输入 | 每个用户一个 | 条 | 8-16位数字与字母混合，区分大小写 |
| 昵称 | 用户输入 | 每个用户一个 | 条 | 32字节（16个中文字符或32个英文字符） |

**3. 处理**

A. 输入数据的有效性检测
- 对手机号检测：验证是否为11位中国大陆手机号格式
- 对邮箱检测：验证是否符合邮箱格式规范
- 对密码检测：验证是否为8-16位且包含数字和字母
- 检测手机号/邮箱是否已被注册

B. 操作的确切次序
```
用户点击注册 → 输入手机号/邮箱 → 输入密码 → 输入昵称 → 提交
→ 后端验证 → 发送验证码 → 用户输入验证码 → 验证通过 → 创建账户 → 返回JWT令牌
```

C. 对异常情况的回应
- 通信失败：提示用户网络异常，请检查网络连接
- 验证码错误：提示验证码错误，请重新输入或重新发送
- 注册信息不合规：对应字段高亮提示

**4. 输出**

A. 对本功能所有输出数据的详细描述

| 输出项 | 输出位置 | 输出内容 | 错误消息 |
|-------|---------|---------|---------|
| 注册成功反馈 | 注册页面 | "注册成功，正在跳转..." | - |
| 登录成功反馈 | 登录页面 | 返回JWT令牌，跳转首页 | - |
| 手机号格式错误 | 输入框下方 | - | "请输入正确的手机号" |
| 密码格式错误 | 输入框下方 | - | "密码须为8-16位，包含数字和字母" |
| 网络异常 | 页面顶部气泡 | - | "网络异常，请检查网络连接" |

#### 3.2.2 Functional Requirements 2 —— 修改个人资料

**1. 介绍**

用户可在个人中心修改个人资料，包括昵称、头像、性别、绑定手机号、密码等。

**预期的错误输入或非法条件：**
- 昵称超出长度限制：提示昵称最多16个中文字符
- 新密码与当前密码相同：提示新旧密码不能相同
- 两次输入的新密码不一致：提示两次输入的密码不一致
- 手机验证码错误：提示验证码错误

**2. 输入**

| 数据项 | 输入来源 | 有效输入范围 |
|-------|---------|-------------|
| 昵称 | 用户输入 | 32字节（16中文字符或32英文字符） |
| 性别 | 用户选择 | 男/女/未知 |
| 手机号 | 用户输入 | 11位数字 |
| 密码 | 用户输入 | 8-16位数字与字母混合 |
| 头像 | 用户上传 | JPG/PNG，最大2MB，建议256×256 |

**3. 处理**

A. 操作的确切次序
```
进入个人中心 → 点击修改资料 → 修改对应字段 → 点击确认修改
→ 后端验证 → 更新数据库 → 返回修改成功
```

```

→ 输入新手机号 → 发送验证码 → 输入验证码 → 完成修改
```
修改密码流程：
```
点击修改密码 → 输入当前密码 → 输入新密码 → 重复输入新密码
→ 验证当前密码正确性 → 验证新旧密码不同 → 验证两次新密码一致 → 完成修改
```

**4. 输出**

| 输出项 | 输出位置 | 输出内容 |
|-------|---------|---------| 
| 修改成功反馈 | 个人中心 | "个人资料修改成功" |
| 修改失败反馈 | 个人中心 | 对应错误提示信息 |

### 3.3 子功能模块二 —— 作品创作（核心模块 M1）

#### 功能简介

作品创作是整个系统的核心功能模块，用户可通过文本上传、AI角色分析、音色配置、情感标注、合成预览和导出下载等步骤，完成一部专业级多角色有声书的创作。

#### 3.3.1 Functional Requirements 1 —— 文本输入与上传

**1. 介绍**

用户通过多种方式将文本内容输入系统，支持直接粘贴文本、上传TXT文件、以及URL导入等方式。系统对输入文本进行预处理（分句、分段落、格式标准化）。

**预期的错误输入或非法条件：**

1. 文本内容为空：提示请输入文本内容
2. 文本包含敏感内容（涉政/涉黄/涉暴）：提示内容不合规，请修改后重试
3. 文件格式不支持：提示仅支持TXT格式文件
4. 文件大小超限：提示文件大小不能超过10MB
5. 文本字数超限：提示单次最多支持50万字

**2. 输入**

| 数据项 | 输入来源 | 有效输入范围 |
|-------|---------|-------------|
| 文本内容 | 用户粘贴 | 50万字符以内 |
| 文本文件 | 用户上传 | TXT格式，UTF-8编码，最大10MB |
| 文本URL | 用户输入 | 可公开访问的网页URL |
| 作品标题 | 用户输入 | 100字节以内 |

**3. 处理**

```
用户输入文本（粘贴/上传/URL） → 文本预处理 → 敏感内容检测
  ├── 通过检测 → 进入角色分析阶段
  └── 不通过 → 返回错误提示
```

**预处理流程**：
- 编码统一（UTF-8）
- 分句处理（根据标点符号切分）
- 段落识别与保留
- 特殊格式处理（引号、书名号等）

**4. 输出**

| 输出项 | 输出位置 | 输出内容 |
|-------|---------|---------|
| 预处理成功 | 创作页面 | 显示文本预览、字数统计、段落数 |
| 内容违规 | 创作页面 | "内容包含不当信息，请修改后重试" |
| 格式错误 | 创作页面 | "仅支持TXT格式文件，请重新上传" |

#### 3.3.2 Functional Requirements 2 —— AI角色分析

**1. 介绍**

系统调用大模型自动分析输入文本，识别故事中的角色信息（名称、性别、性格、说话风格）、对话轮次拆分、角色关系网络构建。这是整个系统智能能力的核心体现。

**预期的错误输入或非法条件：**

1. 文本中无对话内容：提示未检测到对话内容，将以旁白模式合成
2. 角色数量过多（超过20个）：提示角色过多，建议合并配角或使用通用音色
3. AI分析超时：提示分析超时，建议缩短文本或分章节处理

**2. 输入**

| 数据项 | 输入来源 | 有效输入范围 |
|-------|---------|-------------|
| 预处理后的文本 | 上一步输出 | 已预处理的文本数据 |
| 分析类型选择 | 用户选择 | 自动分析 / 手动标注 |

**3. 处理**

A. 角色识别流程
```
预处理文本 → LLM角色分析
  ├── 识别角色数量、名称
  ├── 分析角色性别、性格标签
  ├── 分析角色关系网络
  └── 识别对话边界
→ 对话轮次拆分
  ├── 识别对话模式（"说："、"："、"「」"等）
  ├── 归属每句话的说话角色
  └── 处理旁白与非对话文本
→ 输出：结构化角色信息 + 对话轮次数据
```

B. 角色分析输出示例（JSON格式）
```json

#### 3.3.3 Functional Requirements 3 —— 音色配置

2. 克隆音色未就绪：提示该音色正在训练中，请等待
3. 角色未分配音色：提示检测到未分配音色的角色，请完成配置

**2. 输入**

| 数据项 | 输入来源 | 有效输入范围 |
|-------|---------|-------------|
| 角色音色分配 | 用户选择 | 每个角色分配一个音色ID |
| 音色类型筛选 | 用户选择 | 男声/女声/童声/方言等 |

**3. 处理**

```
用户查看角色列表 → 系统推荐音色 → 用户选择/调整
  ├── 试听音色样本
  ├── 为每个角色分配音色
  └── 支持按条件筛选音色
→ 确认音色配置
```

**音色分类体系**：

| 分类维度 | 类别 |
|---------|------|
| 按性别 | 男声（低沉/磁性/阳刚/温和）、女声（甜美/知性/活泼/成熟）、特殊（童声/老人声） |

|-------|---------|---------|

#### 3.3.4 Functional Requirements 4 —— 情感演绎

**1. 介绍**

系统自动对每句对话进行情感标注（悲伤、愤怒、喜悦、惊讶、平静等），用户可手动微调情感类型和强度。

**情感控制维度**：

| 情感维度 | 控制参数 | 取值范围 | 说明 |
|---------|---------|---------|------|
| 效价（Valence） | 正负情感 | -1.0 ~ +1.0 | 负面→中性→正面 |
| 唤醒度（Arousal） | 激动程度 | 0.0 ~ 1.0 | 平静→激动 |
| 支配度（Dominance） | 掌控感 | 0.0 ~ 1.0 | 被动→主导 |
| 语速（Speed） | 语速倍数 | 0.5x ~ 2.0x | 慢速→快速 |
| 音调（Pitch） | 音调偏移 | -50% ~ +50% | 低沉→高亢 |
| 停顿（Pause） | 停顿时长 | 0ms ~ 2000ms | 无停顿→长停顿 |

**情感-声音映射表**：

| 情感类别 | 语速 | 音调 | 停顿 | 音量 | 典型场景 |
|---------|------|------|------|------|---------|
| 喜悦 | 1.1x-1.3x | +10%~+20% | 短 | 正常-稍高 | 开心、高兴、兴奋 |
| 悲伤 | 0.8x-0.95x | -10%~-20% | 长 | 稍低 | 哭泣、难过、沮丧 |
| 愤怒 | 1.2x-1.4x | +15%~+30% | 中 | 高 | 发火、质问、斥责 |
| 惊讶 | 1.0x-1.2x | +20%~+40% | 短 | 高 | 震惊、感叹、疑问 |
| 平静 | 0.9x-1.0x | ±0%~±5% | 正常 | 正常 | 叙述、讲解、说明 |
| 恐惧 | 1.3x-1.5x | +20%~+35% | 短/不规律 | 不稳定 | 害怕、颤抖、紧张 |
| 温柔 | 0.85x-0.95x | -5%~-15% | 长 | 稍低 | 安慰、关爱、轻声 |

**2. 输入**

| 数据项 | 输入来源 | 有效输入范围 |
|-------|---------|-------------|
| 自动情感标注结果 | 系统自动生成 | 7维情感控制向量 |
| 手动情感微调 | 用户拖拽调节 | 各维度参数的合法范围 |

**3. 处理**

```
角色配置信息 → 情感自动标注 → 用户预览与微调
  ├── 自动情感分析（上下文感知）
  ├── 手动情感参数调节
  ├── 情感强度控制（微/中/强/极强）
  └── 关键情感点标记
→ 确认情感配置
```

**4. 输出**

| 输出项 | 输出位置 | 输出内容 |
|-------|---------|---------|
| 情感标注结果 | 创作页面 | 每句对话的情感标签、强度、韵律参数 |

#### 3.3.5 Functional Requirements 5 —— 合成与预览

**1. 介绍**

用户确认角色配置和情感标注后，触发语音合成。系统支持片段实时预览（流式输出）和全书后台合成两种模式。

**2. 输入**

| 数据项 | 输入来源 | 有效输入范围 |
|-------|---------|-------------|
| 结构化剧本数据 | 上游模块输出 | 包含角色、音色、情感、文本的完整数据 |
| 合成模式选择 | 用户选择 | 片段预览 / 全书合成 |

**3. 处理**

A. 情感TTS合成流程
```
角色配置 + 情感向量 → 音色加载 → 情感控制向量生成
  → CosyVoice 2.0 情感TTS合成
  → Mel频谱生成 → HiFiNet声码器 → 流式输出
→ 高质量音频片段 + 元数据
```

B. 全书合成流程
```
结构化剧本 → 分块处理（每块500-1000字）
  → 多进程并行合成 → 异步任务队列
  → 音频拼接 → 后期处理
→ 成品音频
```

C. 对异常情况的回应
- 合成失败：自动重试3次，仍失败则标记错误段落并通知用户
- 音色加载失败：回退到默认音色并提示用户
- 合成超时：分块处理，超时块跳过并标记

**4. 输出**

| 输出项 | 输出位置 | 输出内容 |
|-------|---------|---------|
| 流式预览音频 | 创作页面播放器 | 实时合成的音频片段 |
| 合成进度 | 创作页面进度条 | 当前合成进度百分比、剩余时间 |
| 质量评分 | 创作页面 | MOS评分、情感准确度、角色一致性 |

#### 3.3.6 Functional Requirements 6 —— 导出与分享

**1. 介绍**

合成完成后，用户可将成品有声书导出为多种格式，或通过链接/嵌入代码分享。

**2. 输入**

| 数据项 | 输入来源 | 有效输入范围 |
|-------|---------|-------------|

| 导出范围 | 用户选择 | 全书 / 按章节 |
| 比特率 | 用户选择 | 128kbps / 192kbps / 320kbps |

**3. 处理**

```
成品音频 → 格式转换 → 封面图生成 → 元数据嵌入
  ├── 多格式导出（MP3/AAC/WAV）
  ├── 章节拆分
  └── 下载链接生成
→ 分享链接/嵌入代码生成

### 3.4 子功能模块三 —— 声音克隆（M1/M3核心亮点）

#### 功能简介

用户可上传少量音频样本（5-10分钟），系统自动训练生成专属的AI音色，实现"声音克隆"功能。

#### 3.4.1 Functional Requirements 1 —— 音频样本上传与检测

**1. 介绍**

用户上传个人音频样本，系统自动进行质量检测（噪音检测、语音清晰度检测、说话人一致性检测），合格后方可进入克隆训练。

**预期的错误输入或非法条件：**

1. 音频时长不足：提示音频时长不足5分钟，请提供更长的音频样本
2. 背景噪音过大：提示音频信噪比过低，请在安静环境重新录制
3. 检测到多人说话：提示检测到多个说话人，请确保音频中仅包含一人
4. 音频格式不支持：提示仅支持MP3/WAV格式
5. 音频含背景音乐：提示检测到背景音乐，请使用纯人声音频

**2. 输入**

| 数据项 | 输入来源 | 有效输入范围 |
|-------|---------|-------------|
| 音频文件 | 用户上传 | MP3/WAV格式，5-10分钟，16kHz以上采样率，单人朗读无BGM |
| 音频名称 | 用户输入 | 32字节以内 |
| 音色标签 | 用户选择 | 男声/女声/童声等 |

**3. 处理**

```
用户上传音频 → 音频质量检测
  ├── 噪音检测（信噪比评估）
  ├── 语音清晰度检测
  ├── 说话人一致性检测
  └── BGM检测
→ 检测通过 → 进入克隆训练
→ 检测不通过 → 返回错误提示
```

**4. 输出**

| 输出项 | 输出位置 | 输出内容 |
|-------|---------|---------|
| 检测结果 | 声音克隆页面 | 各项检测通过/不通过及具体数值 |
| 检测不通过 | 声音克隆页面 | 具体问题描述和改进建议 |

#### 3.4.2 Functional Requirements 2 —— LoRA克隆训练

**1. 介绍**

系统基于CosyVoice 2.0基础模型，使用用户上传的音频进行LoRA微调训练，生成专属音色模型。

**2. 输入**

| 数据项 | 输入来源 | 有效输入范围 |
|-------|---------|-------------|
| 通过检测的音频 | 上游输出 | 预处理后的高质量音频 |
| 基础模型 | 系统预置 | CosyVoice 2.0 |

**3. 处理**

```
原始音频 → 音频预处理（降噪/采样率统一/VAD）
  → 特征提取（说话人embedding/韵律/情感特征）
  → LoRA微调训练（100-200 epochs，约15-30分钟）
  → 音色模型输出（LoRA权重文件~100MB）
```

**4. 输出**

| 输出项 | 输出位置 | 输出内容 |
|-------|---------|---------|
| 训练进度 | 声音克隆页面 | 训练进度百分比、预计剩余时间 |
| 训练完成 | 声音克隆页面 | 克隆音色模型及评估指标 |

#### 3.4.3 Functional Requirements 3 —— 克隆音色测试与入库

**1. 介绍**

训练完成后，用户可输入测试文本预览克隆音色效果，满意后确认保存至个人音色库。

**2. 输入**

| 数据项 | 输入来源 | 有效输入范围 |
|-------|---------|-------------|
| 测试文本 | 用户输入 | 任意文本 |
| 确认保存 | 用户操作 | 确认/重新训练 |

**3. 处理**

```
输入测试文本 → 加载克隆音色模型 → TTS合成预览
→ 用户确认 → 音色入库（个人音色库）
→ 不满意 → 可重新训练或调整参数
```

**克隆音色质量控制标准**：

| 检查项 | 方法 | 标准 |
|-------|------|------|
| 相似度 | 与原始音频对比 | MOS评分 > 4.0 |
| 自然度 | 合成音频质量评估 | MOS评分 > 4.2 |
| 一致性 | 多段文本音色稳定 | 方差 < 0.1 |
| 清晰度 | 字词错误率检测 | WER < 5% |

**4. 输出**

| 输出项 | 输出位置 | 输出内容 |
|-------|---------|---------|
| 预览音频 | 声音克隆页面 | 克隆音色的测试合成结果 |
| 入库确认 | 声音克隆页面 | "音色已保存，可在创作中使用" |

### 3.8 子功能模块七 —— 音频后期处理（M3）

#### 功能简介

对合成后的原始音频进行专业级后期处理，包括降噪、音量标准化、背景音乐添加等，使成品达到出版级有声书质量标准。

#### 3.8.1 Functional Requirements 1 —— 音频后期处理流水线

**1. 介绍**

系统自动对合成音频执行多阶段后期处理：降噪 → 音量标准化 → 角色音频拼接 → 背景音乐添加 → 章节处理。

**处理流水线**：

```
原始合成音频
  ↓
Stage 1: 降噪处理（环境噪音/呼吸声/电流声消除）
  ↓
Stage 2: 音量标准化（响度标准化-16 LUFS，峰值限制）
  ↓
Stage 3: 角色音频拼接（时间轴对齐，旁白与对话穿插）
  ↓
Stage 4: 背景音乐添加（场景识别，BGM自动匹配，音量自动调整）
  ↓
Stage 5: 章节处理（章节检测分割，静音插入，封面嵌入）
  ↓
输出：成品音频 + 多格式导出
```

**2. 输入**

| 数据项 | 输入来源 | 有效输入范围 |
|-------|---------|-------------|
| 原始合成音频 | TTS合成模块输出 | 分段的语音音频数据 |
| 后期处理选项 | 用户配置 | 是否启用降噪/BGM/标准化 |

**3. 处理**

A. 降噪处理
- 使用RNNoise/MetaVoice算法去除非人声噪音
- 抑制呼吸声和电流声
- 保持人声清晰度

B. 音量标准化
- 采用EBU R128标准，响度标准化至-16 LUFS
- 动态范围压缩（避免过大音量差）
- 峰值限制（防止削波失真）

C. 背景音乐智能匹配

| 场景类型 | BGM特征 | 推荐风格 |
|---------|--------|---------|
| 叙述旁白 | 轻柔、无歌词 | 钢琴曲、轻音乐、自然声 |
| 对话场景 | 适中、不干扰 | 室内乐、爵士乐 |
| 紧张悬疑 | 低沉、节奏感 | 弦乐、合成器音效 |
| 温馨感人 | 柔和、情感丰富 | 弦乐合奏、钢琴 |
| 动作打斗 | 快节奏、强烈 | 打击乐、电子音乐 |
| 悲伤场景 | 低沉、小调 | 大提琴、悲伤钢琴 |

**4. 输出**

| 输出项 | 输出位置 | 输出内容 |
|-------|---------|---------|
| 处理后的成品音频 | 存储服务 | 多章节、多格式的音频文件 |

- 音色模型大小：~100MB（LoRA权重文件）

## 7 软件质量特性

### 7.1 可靠性

- **适应性**：系统架构支持功能扩展，新增业务功能不影响原有系统架构
- **容错性**：AI服务异常时有降级方案，使用备用模型（如Edge-TTS）；网络拥塞、系统崩溃时不造成功能失效，可正常关闭及重启
- **可恢复性**：长时合成任务支持断点续传；网络故障恢复后，系统能正常运行

### 7.2 易用性

- 核心创作流程提供分步引导（Wizard模式），降低使用门槛
- 关键概念（音色、情感）提供可视化说明和示例
- 阻止用户输入非法数据或进行非法操作，前端实时校验
- 对错误输入提供明确、友好的提示信息
- 支持"所见即所得"的实时预览，降低试错成本

### 7.3 安全性

- 用户密码使用bcrypt哈希存储，禁止明文存储
- 使用JWT令牌进行身份认证，令牌有过期时间
- API接口进行权限校验，防止越权访问
- 声音克隆功能需实名认证，克隆音频嵌入隐形水印（用户ID+时间戳）
- 敏感内容（涉政/涉黄/涉暴）自动检测和拦截
- 防止SQL注入、XSS、CSRF等常见Web攻击

### 7.4 可维护性

- 代码采用模块化设计，前后端分离
- 关键业务逻辑编写单元测试
- 提供完整的API文档（Swagger/OpenAPI）
- 实施日志分级记录，便于问题定位
- 管理后台提供系统配置界面，减少人工后台修改

### 7.5 可移植性

- 前端基于Web标准，可运行于主流浏览器
- 后端基于Python FastAPI，可部署于Linux和Windows
- AI服务支持云API调用和本地部署两种模式
