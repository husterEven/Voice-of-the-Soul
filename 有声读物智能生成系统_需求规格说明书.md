## 摘要

本需求规格说明书面向个人内容创作者，描述了有声读物智能生成系统的功能和性能需求。系统通过场景化模板、预设音色库、智能文本预处理和音频合成技术，提供"选择场景 → 粘贴文本 → 一键生成 → 快速导出"的极简有声内容制作流程，大幅降低有声内容制作门槛与成本。

## 2 总体概述

### 2.1 软件概述

#### 2.1.1 项目介绍

在数字内容创作蓬勃发展的今天，有声读物已成为内容创作者扩大影响力、实现内容变现的重要渠道。然而个人创作者面临**制作门槛高、配音成本高、操作复杂、个性化不足、效率低下**等核心痛点。

本项目面向自媒体创作者、知识付费讲师、播客主播、教育内容生产者等用户群体，提供**简单易用、快速高效**的AI配音解决方案。用户无需专业设备与配音经验，通过Web端上传文本即可一键生成专业级有声内容。

核心价值主张：**一键配音，让创作更简单**。

#### 项目属性概览

| 项目属性 | 详情 |
|---------|------|
| **项目名称** | 面向内容创作者的个性化有声读物智能生成系统 |
| **核心技术方向** | 场景化TTS / 预设音色库 / 文本智能分段 / 一键配音 |
| **目标用户** | 自媒体创作者、知识付费讲师、播客主播、教育内容生产者 |
| **平台形态** | Web端（创作者工作台）+ API服务层（AI能力输出） |
| **开发周期** | 4周（约20个工作日） |

**各子系统功能概述：**

- **M1-用户端**：直接面向终端用户的Web交互界面，涵盖从场景选择、文本输入、音色配置到最终导出下载的完整创作流程，以及作品管理与个性化设置。
- **M2-AI服务层**：系统的核心智能能力支撑，提供文本预处理、语音合成、情感处理、音频合成四大引擎，是系统智能化处理的中枢。
- **M3-基础支撑**：为整个系统提供用户认证、文件存储、异步任务调度、缓存加速和运维监控等基础能力保障。

### 2.3 用户特征

本系统的目标用户及其特征如下：

| 用户类型 | 特征描述 | 核心需求 |
|---------|---------|---------|
| **自媒体创作者** | 有内容创作能力，缺乏配音设备和经验，追求快速产出 | 简单易用、快速生成、音色丰富 |
| **知识付费讲师** | 需要专业音质的课程录音，注重清晰度和专业感 | 专业音色、语速可调、音质清晰 |
| **播客主播** | 有固定节目需求，需要自然流畅的对话感配音 | 自然音色、场景模板、批量生成 |
| **教育内容生产者** | 需要大量标准化教育音频，注重成本与效率 | 低成本、高效率、批量处理 |
| **普通内容爱好者** | 偶尔使用，技术能力有限 | 极简操作、零门槛上手 |

用户共性特征：缺少专业录音设备、无配音专业训练、追求高效低成本的配音解决方案、对操作复杂度敏感。

#### 3.2.1 场景选择

##### 1 介绍

场景选择是创作流程的核心入口。用户根据内容类型快速选择场景模式，系统自动加载推荐音色和参数配置，降低决策成本。系统提供4种预设场景，覆盖主流有声内容创作需求。

功能需求列表：

| 需求ID | 需求描述 | 优先级 |
|--------|---------|--------|
| SC-001 | 系统应提供4种预设场景模式供用户选择 | 必须的 |
| SC-002 | 选择场景后，系统应自动推荐默认音色和参数配置 | 必须的 |
| SC-003 | 用户应可在推荐基础上自定义调整所有参数 | 必须的 |
| SC-004 | 场景切换时，系统应保留用户已输入的文本内容 | 重要的 |
| SC-005 | 每个场景应展示简要说明和适用内容示例 | 最好有的 |

##### 2 输入

| 输入项 | 描述 |
|--------|------|
| **输入来源** | 用户在Web页面上点击场景卡片进行选择 |
| **输入数量** | 单次创作选择1个场景模式 |
| **可选值** | "播客模式"、"知识讲解"、"故事讲述"、"情感朗读" 四选一 |
| **时间要求** | 即时响应，切换场景<200ms完成配置加载 |
| **默认值** | 未选择场景时默认为空，提示用户先选择场景 |

##### 3 处理

```
用户进入创作页面
    ↓
系统展示4个场景卡片（含图标、名称、简介）
    ↓
用户点击选择场景模式
    ↓
系统根据场景类型查询推荐配置：
    ├── 播客模式 → 自然中性音色，语速1.0x，情感：平静
    ├── 知识讲解 → 清晰专业音色，语速0.9x，情感：平静
    ├── 故事讲述 → 感染力音色，语速1.0x，情感：开心
    └── 情感朗读 → 温暖动人音色，语速0.85x，情感：悲伤/激动
    ↓
系统自动加载推荐音色列表、默认语速/音调、默认情感标签
    ↓
同步更新创作工作台各区域状态
    ↓
用户可保留推荐配置或手动调整
```

**场景模式与默认配置映射：**

| 场景模式 | 适用内容 | 默认音色风格 | 默认语速 | 默认情感 |
|---------|---------|-------------|---------|---------|
| 播客模式 | 播客节目、访谈、对话 | 自然亲切中性音色 | 1.0x | 平静 |
| 知识讲解 | 课程讲解、知识分享、教程 | 清晰专业音色 | 0.9x | 平静 |
| 故事讲述 | 小说朗读、故事讲述 | 富有感染力音色 | 1.0x | 开心 |
| 情感朗读 | 诗歌朗诵、情感文章 | 温暖动人音色 | 0.85x | 悲伤/激动 |

**异常处理：**
- 如果默认推荐音色因故不可用（如被管理员删除），系统应自动降级为通用音色，并提示用户
- 配置加载超时（>3s），应显示加载失败提示，允许用户手动配置

##### 4 输出

| 输出项 | 描述 |
|--------|------|
| **输出目标** | Web前端创作工作台各组件状态更新 |
| **输出内容** | 选中的场景标识、推荐音色列表、默认语速、默认音调、默认情感标签 |
| **时序** | 用户点击后即时更新UI状态 |
| **错误消息** | "场景配置加载失败，请手动选择音色和参数" |

#### 3.2.2 创作工作台-文本输入与预处理

##### 1 介绍

文本输入与预处理功能让用户便捷地导入待配音文本，系统自动进行清洗、分段、停顿标注和敏感词检测。支持直接粘贴和文件上传两种输入方式。

功能需求列表：

| 需求ID | 需求描述 | 优先级 |
|--------|---------|--------|
| TI-001 | 系统应支持文本直接粘贴输入，不限字符数 | 必须的 |
| TI-002 | 系统应支持TXT、DOC文件上传导入文本 | 必须的 |
| TI-003 | 系统应自动对文本进行清洗（去空格、统标点、滤特殊字符） | 必须的 |
| TI-004 | 系统应自动对长文本智能分段（优先按段落，长句>50字按逗号拆分） | 必须的 |
| TI-005 | 系统应自动标注停顿标记（逗号300ms/句号600ms/段落1000ms） | 必须的 |
| TI-006 | 系统应检测文本中的敏感词并高亮提示用户 | 重要的 |
| TI-007 | 系统应支持用户手动插入停顿标记 | 最好有的 |
| TI-008 | 文本输入框应显示字符数统计 | 重要的 |
| TI-009 | 系统应限制单次配音文本最大10000字 | 重要的 |

##### 2 输入

| 输入项 | 描述 |
|--------|------|
| **输入来源** | 用户在创作工作台操作：文本框粘贴 或 文件上传按钮 |
| **输入格式** | 纯文本（粘贴）/ TXT文件 / DOC文件 |
| **文本数量** | 粘贴：1-10000字；上传：单文件最大5MB |
| **度量单位** | 字符数（中文字符计1字） |
| **有效输入范围** | 1-10000个有效字符（仅含中文、英文、数字、常用标点） |
| **非法输入处理** | 空文本提示"请输入文本内容"；超限提示"文本超过10000字限制"；不支持格式提示"请上传TXT或DOC格式文件" |

##### 3 处理

**文本输入处理流程：**

```
原始文本输入（粘贴/文件上传）
    ↓
文本清洗（预处理引擎）
    ├── 去除连续多余空格，保留单空格
    ├── 统一标点符号（全角/半角统一为中文标点）
    ├── 过滤特殊字符（如emoji、控制字符）
    └── 去除首尾空白
    ↓
智能分段（预处理引擎）
    ├── 优先按段落（\n\n）分割
    ├── 对单段内长句（>50字）按逗号/分号进一步拆分
    ├── 保持单段最小长度（>10字），避免过碎片化
    └── 标记段落边界索引
    ↓
停顿标注（预处理引擎）
    ├── 逗号/分号 → 短停顿标记 [pause:300ms]
    ├── 句号 → 中停顿标记 [pause:600ms]
    ├── 段落结束 → 长停顿标记 [pause:1000ms]
    ├── 感叹号/问号 → 情感停顿标记 [pause:800ms]
    └── 生成结构化文本 + 停顿标注数组
    ↓
敏感词检测（预处理引擎）
    ├── 基于预置敏感词库匹配
    ├── 高亮显示敏感词位置
    ├── 提示用户确认或修改
    └── 用户确认后允许继续合成（记录日志）
    ↓
输出：结构化文本（分段数组 + 停顿标记）
```

**输入数据的有效性检测：**
- 空文本检测：输入为空时阻止继续操作
- 字数统计：实时显示当前字数/上限
- 格式检测：文件上传时验证扩展名（.txt/.doc）
- 敏感词检测：匹配预设敏感词库

**异常处理：**
- 文件上传失败（网络中断）→ 提示"上传失败，请重试"，允许重新上传或回退到粘贴模式
- 文本清洗后无有效内容 → 提示"文本内容无效，请重新输入"
- 敏感词检测到大量敏感词 → 提示用户，提供替换建议

##### 4 输出

| 输出项 | 描述 |
|--------|------|
| **输出目标** | Web前端文本编辑区域（带高亮标注）；后端传递给语音合成引擎 |
| **输出格式** | 结构化JSON：`[{segment_index, text, pause_after_ms, has_sensitive_word}]` |
| **输出数量** | 1-N个分段（N取决于文本长度和分段策略） |
| **时序** | 用户输入完成后实时预处理（<500ms完成），结果即时展示 |
| **错误消息** | "文本预处理失败，请检查文本内容"; "检测到X处敏感词，请确认后继续" |

| AE-010 | 合成失败应支持自动重试（最多3次） | 必须的 |

##### 2 输入

| 输入项 | 描述 |
|--------|------|
| **输入来源** | 用户在创作工作台点击"片段试听"/"全文生成"按钮 |
| **合成输入** | 结构化文本分段数组 + 选中音色ID + 语速/音调/情感参数 + BGM配置 |
| **导出输入** | 用户选择导出格式（MP3/WAV） |
| **度量单位** | 文本字数（影响合成时长估算） |
| **时间要求** | 片段试听<5s返回；全文生成不强制实时（异步处理） |

##### 3 处理

**完整合成与导出流程：**

```
用户点击"全文生成"
    ↓
系统创建合成任务（状态：processing）
    ├── 保存作品记录到数据库
    └── 返回任务ID，前端进入等待状态
    ↓
异步任务队列处理（Celery Worker）：
    ├── 阶段1：分段合成
    │   ├── 遍历结构化文本分段数组
    │   ├── 每段调用TTS API合成音频片段（最长500字/段）
    │   ├── 应用语速/音调/情感参数
    │   └── 失败自动重试（最多3次）
    ├── 阶段2：音频拼接
    │   ├── 按顺序拼接音频片段
    │   ├── 在各段间插入对应停顿时长
    │   └── 音量均衡处理
    ├── 阶段3：BGM混合（如选择）
    │   ├── 加载选中BGM文件
    │   ├── 调整音量平衡
    │   ├── 应用淡入淡出
    │   └── 混音输出
    └── 阶段4：格式转换与导出
        ├── 生成MP3（128kbps）
        ├── 生成WAV（如用户选择）
        └── 上传至文件存储服务
    ↓
WebSocket推送合成进度和完成状态
    ↓
前端更新状态：completed，展示播放器和下载链接
    ↓
用户可在线播放 或 下载MP3/WAV文件
```

**进度推送粒度：**
- 每完成一段分段合成 → 推送进度百分比
- 拼接完成 → 推送"正在添加背景音乐..."
- 全部完成 → 推送完成状态和文件URL

**异常处理：**
- TTS API超时（单段>30s）：进入重试逻辑（最多3次），3次后标记该段合成失败
- 部分段落合成失败：跳过失败段落，合成其余部分，在失败位置插入静音标记，完成后提示用户"X个段落合成失败"
- 全部段落失败：标记任务失败（status: failed），提示用户"合成失败，请稍后重试或更换音色"
- BGM混合失败：降级为纯语音输出，不阻断流程
- 后台服务宕机：任务队列持久化，服务恢复后继续处理

##### 4 输出

| 输出项 | 描述 |
|--------|------|
| **输出目标** | 文件存储服务（OSS/本地）；Web前端播放器与下载链接 |
| **输出格式** | MP3（128kbps, 44.1kHz, 立体声）；WAV（可选，无损格式） |
| **输出数量** | 每个任务1个音频文件 + 1个封面图（可选） |
| **文件体积** | MP3约1MB/分钟（128kbps）；WAV约10MB/分钟 |
| **时序** | 任务提交即时响应；片段试听<5秒；全文生成取决于文本长度（预估速率：1000字/分钟） |
| **有效输出范围** | MP3文件大小：1KB-100MB；时长：1秒-60分钟 |
| **错误消息** | "合成失败，错误码：XXX，请稍后重试"；"部分段落合成失败，请查看详情" |

#### 3.3.1 文本预处理引擎

##### 1 介绍

文本预处理引擎负责将原始文本转换为结构化的、标注了停顿信息的文本数据，供语音合成引擎使用。功能包括文本清洗、智能分段、停顿标注和敏感词检测。

##### 2 输入

| 输入项 | 描述 |
|--------|------|
| **输入来源** | 前端API请求 / 创作工作台提交 |
| **输入格式** | JSON: `{"raw_text": "原始文本内容", "options": {"max_chars_per_segment": 500}}` |
| **输入数量** | 每次请求1个文本；文本长度1-10000字 |
| **有效输入范围** | 非空字符串，仅含中文、英文、数字、常用标点符号 |
| **时间要求** | 预处理应在500ms内完成（常规长度文本） |

##### 3 处理

```
接收原始文本
    ↓
步骤1：文本清洗
    ├── 正则匹配去除连续多余空格
    ├── 全角/半角标点统一（统一为中文全角标点）
    ├── 过滤特殊不可见字符和emoji
    └── 去除首尾空白字符
    ↓
步骤2：智能分段
    ├── 优先按双换行(段落)分割
    ├── 单段内长句（中文字数>50）按逗号/分号/冒号拆分
    ├── 保持单段最小字符数>10（避免过碎片化）
    └── 记录每段在原文本中的起止位置
    ↓
步骤3：停顿标注
    ├── 段落结尾 → pause: 1000ms
    ├── 句号结尾 → pause: 600ms
    ├── 问号/感叹号结尾 → pause: 800ms
    ├── 逗号/分号/冒号结尾 → pause: 300ms
    └── 其他结尾 → pause: 400ms（默认）
    ↓
步骤4：敏感词检测
    ├── 基于预设敏感词库进行AC自动机多模式匹配
    ├── 返回敏感词位置列表
    └── 标记"has_sensitive: true"段落
    ↓
输出：结构化JSON
```

**算法说明：**
- **分段算法**：贪心分段策略，优先保持段落完整性，仅在超长句时拆分
- **停顿时长**：基于语言学规则的固定时长映射
- **敏感词检测**：AC自动机（Aho-Corasick）多模式匹配算法，O(n+m)时间复杂度

**异常处理：**
- 文本清洗后无有效内容 → 返回错误码 `TEXT_EMPTY_AFTER_CLEAN`
- 敏感词数量超过阈值（>20处）→ 标记整篇为"敏感内容"，需用户确认
- 输入文本超过字数限制 → 返回错误码 `TEXT_TOO_LONG`

##### 4 输出

```json
{
  "segments": [
    {
      "index": 0,
      "text": "处理后的段落文本",
      "char_count": 45,
      "pause_after_ms": 600,
      "has_sensitive": false,
      "sensitive_words": []
    }
  ],
  "total_segments": 5,
  "total_chars": 250,
  "has_any_sensitive": false,
  "warnings": []
}
```

| 输出项 | 描述 |
|--------|------|
| **输出目标** | 返回给前端展示；传递给语音合成引擎 |
| **错误消息** | `{"error": "TEXT_EMPTY_AFTER_CLEAN", "message": "文本清洗后无有效内容"}` |

#### 3.3.2 语音合成引擎

##### 1 介绍

语音合成引擎封装TTS服务调用，将分段文本转换为语音音频片段。首选Edge-TTS（微软免费TTS服务），备选阿里云基础TTS。

##### 2 输入

| 输入项 | 描述 |
|--------|------|
| **输入来源** | 音频合成引擎调用 / API直接调用 |
| **输入格式** | JSON：`{"segments": [...], "voice_id": "zh-CN-XiaoxiaoNeural", "speed": 1.0, "pitch": "normal"}` |
| **voice_id** | TTS服务对应的音色标识（如Edge-TTS的`zh-CN-YunxiNeural`） |
| **speed** | 语速倍率，范围0.8-1.5 |
| **pitch** | 音调，可选值：low/normal/high |
| **时间要求** | 单段500字应在10秒内完成合成 |

##### 3 处理

```
接收合成请求（单段文本 + 语音参数）
    ↓
选择TTS服务提供商（优先级：Edge-TTS > 阿里云TTS）
    ↓
调用TTS API：
    ├── Edge-TTS：通过edge-tts库调用微软Edge TTS服务
    │   └── 参数：voice=voice_id, rate=(speed调整), pitch=(pitch映射)
    ├── 阿里云TTS（备选）：通过DashScope SDK调用
    │   └── 参数：voice=voice_id, speech_rate=speed, pitch_rate=pitch映射
    ↓
获取音频流
    ├── 接收二进制音频数据（MP3/WAV格式）
    └── 保存为临时音频片段文件
    ↓
返回音频片段路径/字节流
```

**TTS方案对比：**

| 方案 | 成本 | 质量 | 稳定性 | 优先级 |
|-----|------|------|--------|--------|
| Edge-TTS | 免费 | ⭐⭐⭐⭐ | ⭐⭐⭐⭐ | 首选 |
| 阿里云基础TTS | 低 | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐⭐ | 备选（降级） |

**语速参数映射（Edge-TTS rate）：**
- 0.8x → "-20%"
- 0.9x → "-10%"
- 1.0x → "+0%"
- 1.1x → "+10%"
- 1.2x → "+20%"
- 1.5x → "+50%"

**常见Edge-TTS中文音色：**
- zh-CN-XiaoxiaoNeural（女声-温柔）
- zh-CN-YunxiNeural（男声-磁性）
- zh-CN-YunjianNeural（男声-活力）
- zh-CN-XiaoyiNeural（女声-清脆）
- zh-CN-YunyangNeural（男声-标准播音）
- zh-CN-XiaochenNeural（女声-知性）

**异常处理：**
- Edge-TTS连接超时（>30s）：触发重试（最多3次，间隔1s/2s/4s）
- 重试3次仍失败：切换至阿里云TTS备选方案
- 备选方案也失败：返回失败状态 `TTS_SERVICE_UNAVAILABLE`
- 返回空音频数据：重试，若持续失败标记该段为合成失败

##### 4 输出

| 输出项 | 描述 |
|--------|------|
| **输出目标** | 音频合成引擎（下一处理环节）；临时文件存储 |
| **输出格式** | 音频二进制数据（MP3临时文件） |
| **输出数量** | 每段1个音频片段文件 |
| **错误消息** | `{"error": "TTS_SERVICE_UNAVAILABLE", "message": "语音合成服务暂时不可用，请稍后重试"}` |

#### 3.3.3 情感处理引擎

##### 1 介绍

情感处理引擎基于规则对文本进行简单情感识别，将用户选择的情感标签映射为TTS参数调整策略，实现基础的语调变化。

##### 2 输入

| 输入项 | 描述 |
|--------|------|
| **输入来源** | 音频合成引擎调用 |
| **输入格式** | `{"text": "段落文本", "emotion_tag": "happy"}` |
| **emotion_tag** | 平静/开心/悲伤/激动，四选一 |

##### 3 处理

**基于规则的情感识别：**
- 对文本进行关键词匹配（如"哈哈"→开心，"呜呜"→悲伤，"太棒了"→激动）
- 如果文本情感与用户标签冲突，以用户标签为准（用户意图优先）
- 无明确情感词时，按用户标签默认处理

**情感-参数映射（输出给TTS引擎）：**

| 情感标签 | 语速调整 | 音调调整 | TTS rate | TTS pitch |
|---------|---------|---------|---------|-----------|
| 平静(calm) | 1.0x | 标准 | +0% | +0Hz |
| 开心(happy) | 1.1x | 明亮 | +10% | +10Hz |
| 悲伤(sad) | 0.9x | 低沉 | -10% | -10Hz |
| 激动(excited) | 1.2x | 明亮 | +20% | +15Hz |

##### 4 输出

| 输出项 | 描述 |
|--------|------|
| **输出目标** | TTS合成参数调整 |
| **输出格式** | `{"speed": 1.1, "pitch": "high", "rate_adjustment": "+10%", "pitch_adjustment": "+10Hz"}` |

#### 3.3.4 音频合成引擎

##### 1 介绍

音频合成引擎是整个配音流程的核心编排者，负责协调文本预处理、情感处理、语音合成和音频后处理，最终生成成品音频文件。

##### 2 输入

音频合成引擎接收完整的创作配置作为输入。输入格式如下：

| 输入项 | 描述 |
|--------|------|
| **输入来源** | Celery任务队列（异步）或API同步调用 |
| **raw_content** | 原始文本内容 |
| **voice_id** | 选中的音色标识 |
| **speed** | 语速倍率 (0.8-1.5) |
| **pitch** | 音调 (low/normal/high) |
| **emotion** | 情感标签 (calm/happy/sad/excited) |
| **bgm_type** | BGM氛围类型 (relax/warm/tense/formal/null) |
| **bgm_volume** | BGM音量百分比 (0-100) |
| **user_id** | 用户ID |
| **work_id** | 作品ID |

##### 3 处理

**完整音频合成流程（pipeline）：**

```
1. 接收合成任务
    ↓
2. 调用文本预处理引擎
    ├── 输入：raw_content
    └── 输出：结构化分段数组 segments[]
    ↓
3. 遍历分段，逐段/批量合成：
    ├── 对每段文本调用情感处理引擎（获取适配参数）
    ├── 对每段文本调用语音合成引擎（生成音频片段）
    │   └── 参数：voice_id + speed + pitch + emotion参数
    ├── 每段合成时推送进度 (WebSocket)
    └── 收集所有音频片段路径
    ↓
4. 音频拼接（pydub + FFmpeg）
    ├── 按顺序加载音频片段
    ├── 在各片段间插入对应停顿时长的静音段
    │   ├── 300ms → 生成300ms静音片段
    │   ├── 600ms → 生成600ms静音片段
    │   └── 1000ms → 生成1000ms静音片段
    ├── 拼接所有片段为完整音频
    └── 应用音量均衡（normalization: -3dB峰值）
    ↓
5. 背景音乐混合（如选择BGM）
    ├── 加载BGM文件
    ├── 调整BGM音量为 (volume/100) × (-18dB)
    ├── BGM截取至配音时长（不足则循环）
    ├── 开头2秒淡入 + 结尾3秒淡出
    ├── 语音轨与BGM轨混合
    └── 输出混音后文件
    ↓
6. 格式转换
    ├── 默认输出MP3（128kbps, 44.1kHz）
    ├── 可选输出WAV（无损16bit PCM）
    └── 上传至文件存储服务
    ↓
7. 更新数据库：作品状态→completed, 文件URL, 时长
    ↓
8. WebSocket推送完成状态
```

**技术栈：**
- 音频拼接与处理：pydub + FFmpeg
- BGM库：预置免版权音乐文件（4种氛围 × 3首备选）
- 音量均衡：pydub的normalize方法

**异常处理：**
- 音频片段文件缺失：跳过该段，继续拼接，末尾记录警告
- 全部片段缺失：标记任务失败
- BGM文件缺失：跳过BGM添加，仅输出语音
- 格式转换失败：尝试降级方案（WAV→MP3失败则仅保留WAV）
- 文件上传失败：本地保留副本，提示用户

##### 4 输出

| 输出项 | 描述 |
|--------|------|
| **输出目标** | 文件存储服务URL；数据库作品记录更新；WebSocket推送 |
| **输出格式** | 成品MP3文件（128kbps）；成品WAV文件（可选） |
| **元数据** | 文件URL、时长（秒）、文件大小（字节）、格式 |
| **错误消息** | 任务级：`{"work_id": "xxx", "status": "failed", "error": "ALL_SEGMENTS_FAILED"}` |
